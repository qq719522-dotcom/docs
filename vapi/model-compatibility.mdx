## 模型兼容

iqqz API 完全兼容 OpenAI 风格接口，适合直接接入支持 OpenAI 协议的应用。

> 提示：所有聊天模型都建议优先使用 OpenAI 官方库和请求格式；非 OpenAI 模型也可以按同样方式接入。

|聊天接口类型|接口地址|接口说明|
|--|--|--|
|OpenAI 通用格式|`/v1/chat/completions`|可调用本站大部分聊天模型|
|OpenAI 新格式|`/v1/responses`|仅部分 OpenAI 新模型支持，通常建议优先使用通用格式|
|Claude 原生格式|`/v1/messages`|用于支持 Claude 原生格式的模型|
|Gemini 原生格式|`/v1beta/models/...`|用于 Gemini 模型，参数名请使用小驼峰命名|

## 身份认证

所有请求都需要在 HTTP Header 中带上 API Key。

|数据格式|Header 传参|
|--|--|
|OpenAI 通用格式|`Authorization: Bearer sk-xxx`|
|Claude 原生格式|`x-api-key: sk-xxx`<br/>`anthropic-version: 2023-06-01`|
|Gemini 原生格式|`x-goog-api-key: sk-xxx`|

## API 地址

请在 `iqqz 控制台` 中查看你当前站点对应的 API 地址。

> 不要把 API 地址和 Key 配错，否则请求会直接失败。
